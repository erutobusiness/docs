# デザインにおける「線（区切り線）」の使用ガイドライン

## 概要

本ドキュメントでは、Twitterユーザー @mmmiyama_D（ミヤマ氏）によって提唱された**「思考停止で線を入れない」**というデザイン原則についてまとめる。

「線（罫線、枠線）」は要素を区切るための便利なツールだが、安易に使いすぎると画面がうるさくなり、情報の構造がかえって見えにくくなる場合がある。
現代的なデザインでは、線に頼らず**「余白」**や**「近接（グルーピング）」**を活用することが推奨される。

## 原則：「その線、いるかな？」

### 線を使うデメリット

- 視覚的なノイズが増える
- 情報の優先順位が曖昧になる
- 「窮屈さ」や「古臭さ」を感じさせる原因になる

### 線を使わない代替手段

1. **余白 (White Space)**: 十分なスペースを空けることで、要素の区切りを表現する
2. **背景色 (Background Color)**: 異なる要素群に淡淡い背景色を敷くことで、エリアを区切る
3. **近接 (Proximity)**: 関連する情報を近づけ、関連しない情報を遠ざけることで、グループを視覚化する
4. **整列 (Alignment)**: 要素の端を揃えることで、見えない線（グリッド）を感じさせる

## 実践例 (Lv別)

### Lv1: カード型レイアウト

- **【Before】線あり**: 各アイテムを四角い枠線で囲む
  - → 情報が分断され、閉塞感がある
- **【After】線なし**: 枠線を消し、余白でアイテム間の距離をとる
  必要に応じて薄いドロップシャドウや背景色を使用
  - → すっきりと開放的で、コンテンツに目がいく

### Lv2: リスト型レイアウト

- **【Before】線あり**: 各行の下に区切り線を入れる
  - → 線が目立ちすぎて、文字情報への集中を妨げる
- **【After】線なし**: 行間を広めに取るだけで十分区別できる
  - → テキストのリズムが生まれ、読みやすくなる

### Lv3: 表組み (Table)

- **【Before】格子状の罫線**: 全てのセルを線で囲む
  - → エクセルのような「データ」感が強く、読みにくい
- **【After】横線のみ / 背景色**: 縦線をなくし、横線のみにする
  または、1行おきに背景色を変える（ゼブラ柄）
  - → 視線の横移動がスムーズになる

### Lv5: 複雑な情報群 (名刺・プロフィール)

- **【Before】枠線で区画整理**: 写真、名前、連絡先などをそれぞれ枠で囲う
  - → 情報のつながりが断たれ、バラバラな印象
- **【After】近接と整列**: 関連する情報を近づけて配置し、グループ化する
  左揃えなどで整列させる
  - → 直感的に情報のまとまりが理解でき、洗練された印象になる
