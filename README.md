# docs

## md

ドキュメントコンテンツを格納するディレクトリ。
各コンテンツの詳細は [md/README.md](./md/README.md) を参照。

## Biome の設定について

このプロジェクトでは、コードの品質維持とフォーマットの統一のために [Biome](https://biomejs.dev/) を使用しています。
Biome はコードフォーマッターとリンターを 1 つのツールに統合したものです。

### 使用方法

プロジェクトルートで以下のコマンドを実行できます：

```bash
# コードをフォーマットする
npm run format

# コードをチェックする
npm run lint

# 自動修正可能な問題を修正する
npm run lint:fix
```

### 設定ファイル

- `biome.json`: Biome の設定ファイル（プロジェクトルートに配置）
- `.gitignore`: Biome が無視するファイルパターン（`useIgnoreFile: true` 設定で使用）

### 注意事項

- `md` ディレクトリ下のマークダウンファイルを修正する場合は `00 - rules.md` を参照・更新してください
- 画像ファイル (`img` ディレクトリ) と PDF ファイル (`pdf` ディレクトリ) は Biome の対象外です

## Textlint の設定について

このプロジェクトでは、ドキュメントの品質向上のため `textlint` を導入しており、プロジェクト独自のカスタム設定を利用しています。

### カスタムフィルタとルール

本文と注釈（脚注）で文体ルールを切り替えるために、以下のカスタム設定を作成・利用しています。

1.  **`textlint-filters/footnote`**
    - **役割**: 注釈（Footnote）部分において、標準の `no-mix-dearu-desumasu`（常体・敬体混在チェック）などのルールを**無効化（除外）**します。
    - **目的**: 本文は「である」調、注釈は「ですます」調といった使い分けを実現するための準備です。

2.  **`textlint-rules/footnote-dearu-desumasu`**
    - **役割**: 注釈（Footnote）部分**専用**の文体チェックを行います。
    - **仕様**: 注釈内が「ですます」調で統一されているかを監視します。
    - **目的**: 標準ルールを除外した注釈部分に対して、別途「ですます」調を強制するために使用します。

これらの設定により、「本文は常体、注釈は敬体」という高度なスタイルガイドラインを機械的にチェック可能にしています。
