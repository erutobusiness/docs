# 厳選・育成 ⟷ 実装フェーズ

## 対応関係

| ポケモン対戦 | エンジニアリング |
|------------|----------------|
| 個体厳選 | コード実装 |
| 努力値振り | パフォーマンスチューニング |
| 技構成決定 | API/機能設計 |
| 持ち物選択 | ライブラリ・ツール選定 |

## 共通する特徴

### 設計の具体化フェーズ

頭の中にあった構想を、実際の形にする工程。

- **ポケモン**:「ようきAS252振り」を実際に育成
- **エンジニア**: 設計書をコードに落とし込む

### やり直しコストが高い

最も時間と労力がかかるフェーズ。安易にやり直せない。

- **ポケモン**: レベル上げ、努力値振り直しの手間
- **エンジニア**: 実装し直し、テスト書き直し、ドキュメント更新...

### 細部へのこだわりが成果を左右

1の差が、勝敗を分けることがある。

- **ポケモン**:「S1振り」「HP16n-1調整」
- **エンジニア**:「レイテンシ10ms改善」「メモリ使用量の最適化」
