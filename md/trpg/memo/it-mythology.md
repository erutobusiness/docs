# IT技術と神話の親和性

IT技術とクトゥルフ神話の親和性についてのメモ。
神話由来のIT用語も多く、シナリオのフレーバーとして活用できる。

## Squid とクトゥルフ / プロキシとリバプロ

### 名前の符合

- **Squid**（プロキシサーバーソフトウェア）= イカ
  - クトゥルフの触手、海洋的イメージとの一致
  - 「イカのソフトウェアを使ってたら本物が来た」的な導入

### プロキシ = 代理・仲介者

- Proxy の本来の意味は「代理人」
- カルトにおける「神との仲介者」としての司祭
- 人間が直接外なる神と接触できないから「代理」を通す
  - プロキシサーバー = 人間界と異次元の間に立つ存在？

### リバースプロキシ = 外から内への経路

- 通常のプロキシ: 内部 → 外部への接続を代理
- リバースプロキシ: 外部 → 内部への接続を受け入れる
- **シナリオ的解釈**: リバプロの設定ミスで「外なる神々」からの接続を受け入れてしまう
  - 「ポートを開けたら、別の何かが繋がってきた」

### シナリオアイデア断片

- 舞台: IT企業、データセンター、サーバールーム
- 導入例:
  - 深夜のサーバーメンテナンス中に異常なトラフィックを検知
  - 誰も設定していないはずのプロキシ経由で通信が発生
  - ログに残る意味不明な文字列 → 実は古代の言語
- 雰囲気:
  - 空調の唸り、サーバーラックの緑のLED、無機質な空間
  - 「誰もいないはずなのに、何かがアクセスしている」

### 技術用語 → 神話的読み替え
