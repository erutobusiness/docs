# Column 3: AIとの対話のコツ

> AIコーディングアシスタントを使いこなすには、「どう伝えるか」が重要です。

## プロンプトエンジニアリングの基本

### 1. 具体的に伝える

```
❌ 「いい感じにして」
✅ 「エラーハンドリングを追加して、失敗時はnullを返すようにして」
```

### 2. 文脈を与える

```
❌ 「この関数を直して」
✅ 「このReactコンポーネントで、useEffectの依存配列が
    空なのでstateが更新されても再実行されません。
    countが変わったら再実行するようにしてください」
```

### 3. 例を示す

```
入力: [1, 2, 3]
期待する出力: [2, 4, 6]

このような変換を行う関数を書いてください。
```

## Claude Code特有のコツ

### ファイルを指定する

```
src/utils/helper.ts を読んで、改善点を教えて
```

### タスクを分解して伝える

```
1. まずテストを書いて
2. 次に実装して
3. 最後にテストを実行して
```

### 確認を求める

```
実行する前に計画を見せて
```

## Google Antigravity特有のコツ

### ゴール（目的）を共有する

Antigravityは自律的に計画（Plan）を立てる能力があります。
細かな手順指示よりも「何を実現したいか」「なぜやるのか」を伝えると効果的です。

```
❌ 「ファイルAを開いて修正して」
✅ 「この機能のパフォーマンスを改善したい。計測結果はこれです」
```

### アーティファクトで認識を合わせる

複雑なタスクでは `implementation_plan.md` や `task.md` などのArtifactsが自動生成されます。
これらをレビューすることで、AIの進行方向を微調整できます。

```
計画を見て、方針が合っていれば「OK」と伝えるだけで、
あとはAgentic Modeが自律的に作業を進めてくれます。
```

### Agentic Modeを理解する

タスク実行中（Agentic Mode）は、ユーザーへの報告（`notify_user`）があるまで自律的に作業します。

```
進捗はTask Statusや、生成されるArtifactsで随時確認できます。
```

## やってはいけないこと

- 曖昧な指示を繰り返す
- 一度に大量のことを頼む
- AIの出力を確認せずに使う
