# ニコニコ組曲カラオケ追加タスク: Claude Code vs Google Antigravity (Gemini) 比較

## 元の指示

> ニコニコ組曲にある曲を karaoke に追加したい

追加する曲を選ぶか聞いたところ、以下の回答:

> 追加予定のリストとして、XX_kumikyoku_claude.md として記載して。

## 使用モデル

- **Claude Code**: Claude Opus 4.5 (`claude-opus-4-5-20251101`)
- **Google Antigravity**: Gemini 3 Pro (`gemini-3-pro`)

## 各AIの対応

### Claude Code / Claude Opus 4.5 (`XX_kumikyoku_claude.md`)

- 組曲『ニコニコ動画』の全33曲をWeb検索（ニコニコ大百科）で確認
- 既存のkaraokeファイル6つを読み込み、登録済みの6曲を特定
- 既登録曲を除外し、未登録の27曲のみを「追加予定リスト」として記載
- 独自のカテゴリ分け（東方アレンジ、ニコニコ系など）で整理

**指示への忠実さ**: 「追加予定のリスト」という依頼に対し、追加すべき曲だけを記載しており忠実

**弱点**:

- 一部の出典が不正確（「森のキノコにご用心」をニコニコ系に分類 → 実際はスーパーマリオRPG）
- アーティスト名の精度が低い（もじぴったんの「nana project」→ 正しくは「古原奈々」）
- 東方アレンジのサークル名が未記載
- 既存ファイルへの振り分け先が不明確

### Google Antigravity / Gemini 3 Pro (`XX_kumikyoku_antigravity.md`)

- 既存のkaraokeファイル構造（`04_anime.md`, `05_game.md` など）に合わせて振り分け先を明示
- 既登録曲も `[済]` タグ付きでリスト内に残し、組曲全体の対応状況を一覧可能に
- 既存ファイルの誤字（武装錬金「真っ赤な近い」→「真赤な誓い」）を指摘
- アーティスト名・出典の正確性が高い（サークル名、ゲーム正式タイトルなど）

**指示への忠実さ**: 「追加予定リスト」の範囲を超え、既登録曲や振り分け先情報も含めており、指示以上の対応

**弱点**:

- 遊☆戯☆王（クリティウスの牙）を含めていない（意図的判断の可能性あり）
- 「さくら」が含まれていない
